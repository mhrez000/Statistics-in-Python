Normal (Gaussian) Distribution

The central distribution of statistics — 68-95-99.7 rule
Notation
X ~ N(μ, σ²)
Parameters
μ∈ℝ, σ>0
E(X)
μ
Var(X)
σ²
Skewness
0 (symmetric)
Kurtosis
3 (mesokurtic)

The Normal distribution is the most important distribution in all of statistics. Its bell-shaped curve is perfectly symmetric around μ. The Central Limit Theorem (CLT) guarantees that the mean of any large sample approaches Normal — regardless of the original distribution. This is why it appears everywhere.

f(x) = (1 / (σ√(2π))) · exp(−(x−μ)² / (2σ²))

68-95-99.7 Rule: ~68% of data within ±1σ, ~95% within ±2σ, ~99.7% within ±3σ. Memorize this — it appears in interviews, papers, and daily DS work constantly.

Real-world: Human heights, IQ scores, measurement errors, residuals in linear regression, log-returns of financial assets (approximately), noise in signals, feature distributions after standardization.
from scipy.stats import norm
import numpy as np

# ─── Normal Distribution ─────────────────────────────────────────
mu, sigma = 170, 10   # Heights in cm
dist = norm(mu, sigma)

print("═══ Normal(μ=170, σ=10) — Human heights ═══")
print(f"P(160 ≤ X ≤ 180) = {dist.cdf(180)-dist.cdf(160):.4f}  ← 68.27% rule (±1σ)")
print(f"P(150 ≤ X ≤ 190) = {dist.cdf(190)-dist.cdf(150):.4f}  ← 95.45% rule (±2σ)")
print(f"P(140 ≤ X ≤ 200) = {dist.cdf(200)-dist.cdf(140):.4f}  ← 99.73% rule (±3σ)")
print()

# 68-95-99.7 rule verified for ANY Normal
print("─── 68-95-99.7 Rule (exact values) ───")
std_norm = norm(0, 1)
for k in [1, 2, 3, 1.96, 2.576]:
    p = std_norm.cdf(k) - std_norm.cdf(-k)
    print(f"  ±{k}σ: {p*100:.4f}% of data")
print()

# Key probability queries with inverse CDF
print("─── Using Inverse CDF for Thresholds ───")
print(f"Top 5% threshold  : {dist.ppf(0.95):.2f} cm")
print(f"Bottom 10% threshold: {dist.ppf(0.10):.2f} cm")
print(f"Middle 80% range  : {dist.ppf(0.10):.2f} — {dist.ppf(0.90):.2f} cm")
print()

# Linear regression residuals should be Normal — how to check
print("─── Checking Normality of Residuals (Shapiro-Wilk test) ───")
from scipy.stats import shapiro
np.random.seed(42)
normal_residuals = np.random.normal(0, 1, 100)
skewed_residuals  = np.random.exponential(1, 100) - 1

stat1, p1 = shapiro(normal_residuals)
stat2, p2 = shapiro(skewed_residuals)
print(f"Normal  residuals: W={stat1:.4f}, p={p1:.4f} → {'Normal ✓' if p1>0.05 else 'Not normal'}")
print(f"Skewed  residuals: W={stat2:.4f}, p={p2:.4f} → {'Normal' if p2>0.05 else 'Not normal ✗'}")
print()

# ASCII shape
print("─── PDF shape ───")
for x in np.arange(145, 196, 5):
    d = dist.pdf(x); bar = '▓'*int(d*400)
    print(f"  {x:3.0f}cm |{bar}")

P(140 ≤ X ≤ 200) = 0.9973 ← 99.73% rule (±3σ)

Crown jewel of statistics: The Central Limit Theorem says sample means follow Normal regardless of original distribution. This justifies t-tests, confidence intervals, linear regression, and many ML algorithms. Always check residuals for normality in linear models.

Standard Normal (Z) Distribution

Normal with μ=0, σ=1 — the universal reference distribution
Notation
Z ~ N(0,1)
E(Z)
0
Var(Z)
1
Z-score
Z=(X-μ)/σ

The Standard Normal is a Normal distribution shifted to have mean 0 and scaled to have std dev 1. Any Normal(μ,σ) can be converted via Z = (X−μ)/σ. This lets you use a single table/function for all Normal distributions. Z-scores tell you how many standard deviations a value is from its mean.

Z = (X − μ) / σ     P(Z ≤ z) = Φ(z)     (Φ is the standard normal CDF)
from scipy.stats import norm
import numpy as np

# ─── Standard Normal ─────────────────────────────────────────────
Z = norm(0, 1)

print("═══ Z-Score: Converting any Normal to Standard Normal ═══")
print("Heights ~ N(170, 10²). A person is 185cm.")
x, mu, sigma = 185, 170, 10
z_score = (x - mu) / sigma
print(f"Z = (185 - 170) / 10 = {z_score:.2f}")
print(f"Interpretation: 185cm is {z_score} standard deviations ABOVE the mean")
print(f"P(height ≤ 185cm) = Φ({z_score}) = {Z.cdf(z_score):.4f}")
print(f"This person is taller than {Z.cdf(z_score)*100:.1f}% of the population")
print()

# Z-table — critical values every data scientist must know
print("─── Critical Z-Values (memorize these!) ───")
critical = {
    '90% CI (two-tailed)': 0.95, '95% CI (two-tailed)': 0.975,
    '99% CI (two-tailed)': 0.995, '5% sig (one-tail)': 0.95,
    '1% sig (one-tail)': 0.99
}
for label, q in critical.items():
    z_c = Z.ppf(q)
    print(f"  {label:<28}: z = {z_c:.4f}")
print()

# Feature standardization in ML (Z-score normalization)
print("─── Feature Standardization for ML ───")
np.random.seed(42)
raw_feature = np.random.normal(5000, 1200, 1000)  # e.g., income
standardized = (raw_feature - raw_feature.mean()) / raw_feature.std()
print(f"Original: mean={raw_feature.mean():.1f}, std={raw_feature.std():.1f}, range=[{raw_feature.min():.0f},{raw_feature.max():.0f}]")
print(f"Z-scored: mean={standardized.mean():.4f},  std={standardized.std():.4f},  range=[{standardized.min():.2f},{standardized.max():.2f}]")
print("After standardization: mean=0, std=1 — gradient descent converges faster!")
▶ Output ═══ Z-Score: Converting any Normal ═══
Z = (185 - 170) / 10 = 1.50
Interpretation: 185cm is 1.5 standard deviations ABOVE the mean
P(height ≤ 185cm) = Φ(1.5) = 0.9332
This person is taller than 93.3% of the population

─── Critical Z-Values ───
90% CI (two-tailed) : z = 1.6449
95% CI (two-tailed) : z = 1.9600 ← THE famous 1.96
99% CI (two-tailed) : z = 2.5758
5% sig (one-tail) : z = 1.6449
1% sig (one-tail) : z = 2.3263

─── Feature Standardization ───
Original: mean=5006.5, std=1199.4, range=[1239,9016]
Z-scored: mean=0.0000, std=1.0000, range=[-3.14,3.34]
After standardization: mean=0, std=1 — gradient descent converges faster!
💡

In data science: Z-score standardization is applied to features before training SVMs, neural networks, logistic regression, and K-means. The z-score is also a one-sample z-test statistic. Φ(z) appears in the Black-Scholes option pricing formula.

Continuous · Equal Probability

Uniform Distribution

Every value in an interval is equally likely
Notation
X ~ U(a,b)
E(X)
(a+b)/2
Var(X)
(b−a)²/12
PDF
1/(b−a) on [a,b]

The Uniform distribution assigns equal probability density to every point in [a, b]. It's the most "uninformative" distribution — used when you have no reason to prefer any value over another. The PDF is a flat horizontal line; the CDF is a straight diagonal line.

f(x) = 1/(b−a)   for a ≤ x ≤ b     F(x) = (x−a)/(b−a)
from scipy.stats import uniform
import numpy as np

# ─── Uniform Distribution ────────────────────────────────────────
a, b = 0, 10
dist = uniform(loc=a, scale=b-a)  # scipy: loc=a, scale=b-a

print("═══ Uniform(0, 10) — Random wait time (0-10 min) ═══")
print(f"E(X)   = (a+b)/2 = {dist.mean():.2f} min")
print(f"Var(X) = (b-a)²/12 = {dist.var():.4f}")
print(f"PDF at any x ∈ [0,10] = {dist.pdf(5):.4f}  ← constant!")
print(f"PDF outside [0,10]   = {dist.pdf(11):.4f}  ← zero outside range")
print()
print(f"P(wait < 3 min)    = {dist.cdf(3):.4f}  = (3-0)/(10-0)")
print(f"P(wait > 7 min)    = {1-dist.cdf(7):.4f}")
print(f"P(4 < wait < 6)    = {dist.cdf(6)-dist.cdf(4):.4f}  (20% of range = 20% prob)")
print()

# Key DS application: random number generation and random search
print("─── Hyperparameter search with Uniform prior ───")
np.random.seed(42)
learning_rates = uniform(loc=1e-5, scale=0.1-1e-5).rvs(5)

In data science: U(0,1) is the foundation of all random sampling and simulation. Used in random search for hyperparameter tuning, Monte Carlo methods, dropout regularization (random mask), train/test splits, and the Probability Integral Transform.

Continuous · Time Between Events

Exponential Distribution

Time between Poisson events — memoryless continuous distribution
Notation
X ~ Exp(λ)
E(X)
1/λ
Var(X)
1/λ²
Memoryless
Yes

The Exponential models the time between events in a Poisson process. If events occur at rate λ per unit time, the waiting time between events is Exp(λ). Like Geometric (its discrete cousin), it has the memoryless property: P(X > t+s | X > t) = P(X > s).

f(x) = λ·e^(−λx)   for x ≥ 0     F(x) = 1 − e^(−λx)     E(X)=1/λ, Var(X)=1/λ²
from scipy.stats import expon, poisson
import numpy as np

# ─── Exponential Distribution ────────────────────────────────────
# Scenario: Server handles 3 requests/sec (λ=3). Time between requests?
lam = 3
dist = expon(scale=1/lam)   # scipy: scale = 1/λ = mean

print("═══ Exponential(λ=3) — Time between server requests ═══")
print(f"E(X) = 1/λ = {dist.mean():.4f} sec between requests")
print(f"Var(X) = 1/λ² = {dist.var():.4f}")
print(f"Std(X) = 1/λ  = {dist.std():.4f}  ← EQUAL to mean (unique property!)")
print()

print(f"P(wait < 0.5s) = {dist.cdf(0.5):.4f}  = 1 - e^(-3×0.5)")
print(f"P(wait > 1.0s) = {1-dist.cdf(1.0):.4f}  = e^(-3×1.0)")
print(f"P(0.2 < wait < 0.8) = {dist.cdf(0.8)-dist.cdf(0.2):.4f}")
print()

# Memoryless property
print("─── Memoryless Property ───")
t, s = 0.5, 0.3
p_conditional = (1-dist.cdf(t+s)) / (1-dist.cdf(t))
p_marginal    = 1 - dist.cdf(s)
print(f"P(X > 0.8 | X > 0.5) = {p_conditional:.4f}")
print(f"P(X > 0.3)           = {p_marginal:.4f}  ← SAME! Memoryless ✓")
print()

# Poisson-Exponential duality
print("─── Poisson-Exponential Duality ───")
print("If events occur at rate λ=3/sec:")
print(f"  Poisson(λ=3): P(0 events in 1s) = {poisson.pmf(0,3):.4f}")
print(f"  Exp(λ=3):     P(wait > 1s)      = {1-dist.cdf(1):.4f}  ← SAME! ✓")
print("Both equal e^(-λ) — the distributions are two sides of the same process")
print()

# Survival analysis application
print("─── Survival Analysis / Reliability Engineering ───")
print("Machine fails at rate λ=0.01 per hour. What is its survival curve?")
lam_fail = 0.01
survival = expon(scale=1/lam_fail)
for t in [10, 50, 100, 200, 500]:
    p_survive = 1 - survival.cdf(t)
    print(f"  P(survive {t:>4} hours) = {p_survive:.4f}")
▶ Output ═══ Exponential(λ=3) ═══
E(X) = 1/λ = 0.3333 sec between requests
Std(X) = 1/λ = 0.3333 ← EQUAL to mean (unique property!)

P(wait < 0.5s) = 0.7769 = 1 - e^(-1.5)
P(wait > 1.0s) = 0.0498 = e^(-3.0)
P(0.2 < wait < 0.8) = 0.4866

─── Memoryless Property ───
P(X > 0.8 | X > 0.5) = 0.4066
P(X > 0.3) = 0.4066 ← SAME! Memoryless ✓

─── Poisson-Exponential Duality ───
Poisson(λ=3): P(0 events in 1s) = 0.0498
Exp(λ=3): P(wait > 1s) = 0.0498 ← SAME! ✓

─── Survival Analysis ───
P(survive 10 hours) = 0.9048
P(survive 100 hours) = 0.3679 ← e^(-1), 37% at mean lifetime
P(survive 500 hours) = 0.0067
💡

In data science: Exponential is used in survival analysis (time-to-event), queuing theory (call center wait times), reliability engineering (time-to-failure), and Poisson process modeling. It's the continuous analog of the Geometric distribution.

Continuous · Right-Skewed

Log-Normal Distribution

When the logarithm of the variable is normally distributed
Notation
X ~ LogN(μ,σ²)
E(X)
e^(μ+σ²/2)
Var(X)
(e^σ²−1)e^(2μ+σ²)
Support
(0, +∞)

If Y = ln(X) is normally distributed with mean μ and std σ, then X follows a Log-Normal. It is always right-skewed and strictly positive. It arises when values are the product of many independent random factors (multiplicative processes), just as Normal arises from sums.

f(x) = (1/(xσ√(2π))) · exp(−(ln x − μ)² / (2σ²))    x > 0

Parameters μ and σ are the mean and std of ln(X), not of X itself. To work with log-normal data: take log → work in normal space → transform back.

from scipy.stats import lognorm, norm
import numpy as np

# ─── Log-Normal Distribution ─────────────────────────────────────
# Scenario: Income distribution. log(income) ~ Normal(mu_log, sigma_log²)
mu_log, sigma_log = 10.5, 0.8  # parameters on log scale
# scipy lognorm: s=sigma, scale=e^mu
dist = lognorm(s=sigma_log, scale=np.exp(mu_log))

print("═══ LogNormal(μ=10.5, σ=0.8) — Income (USD) ═══")
print(f"Parameters on LOG scale: μ={mu_log}, σ={sigma_log}")
print(f"E(Income)   = e^(μ+σ²/2) = ${dist.mean():,.0f}")
print(f"Median income = e^μ       = ${np.exp(mu_log):,.0f}  ← always < mean")
print(f"Std(X)       = ${dist.std():,.0f}")
print(f"Skewness (right-skewed): {dist.stats('s'):.4f}")
print()

print(f"P(income < $20k)  = {dist.cdf(20000):.4f}")
print(f"P(income < $50k)  = {dist.cdf(50000):.4f}")
print(f"P(income > $200k) = {1-dist.cdf(200000):.4f}")
print()

# How to work with log-normal: transform → work in log space
print("─── Working with Log-Normal Data ───")
np.random.seed(42)
incomes = lognorm.rvs(s=sigma_log, scale=np.exp(mu_log), size=1000)
log_incomes = np.log(incomes)

print(f"Raw incomes:     mean=${incomes.mean():,.0f}, median=${np.median(incomes):,.0f}, skew={((incomes-incomes.mean())**3).mean()/incomes.std()**3:.2f}")
print(f"Log(incomes):    mean={log_incomes.mean():.3f}, std={log_incomes.std():.3f} ← normally distributed!")
print()

from scipy.stats import shapiro
_, p_raw = shapiro(incomes[:100])
_, p_log = shapiro(log_incomes[:100])
print(f"Shapiro-Wilk on raw:    p={p_raw:.6f} → {'Normal' if p_raw>0.05 else 'NOT Normal'}")
print(f"Shapiro-Wilk on log():  p={p_log:.4f}  → {'Normal ✓' if p_log>0.05 else 'NOT Normal'}")
print("→ Take the log first, then apply normal-distribution methods!")
▶ Output ═══ LogNormal(μ=10.5, σ=0.8) — Income ═══
E(Income) = e^(μ+σ²/2) = $52,457
Median income = e^μ = $36,316 ← always < mean
Std(X) = $50,956
Skewness (right-skewed): 3.9038

P(income < $20k) = 0.1628
P(income < $50k) = 0.5899
P(income > $200k) = 0.0442

─── Working with Log-Normal Data ───
Raw incomes: mean=$52,218, median=$36,129, skew=3.31
Log(incomes): mean=10.495, std=0.799 ← normally distributed!

Shapiro-Wilk on raw: p=0.000000 → NOT Normal
Shapiro-Wilk on log(): p=0.2312 → Normal ✓
→ Take the log first, then apply normal-distribution methods!

In data science: Income, house prices, stock prices, file sizes, city populations — all Log-Normal. When you log-transform a right-skewed feature in preprocessing, you're acknowledging it's log-normally distributed. Log returns in finance are assumed Normal, making prices Log-Normal.

Continuous · Probabilities & Proportions

Beta Distribution

Distribution over [0,1] — the natural prior for probabilities
Notation
X ~ Beta(α,β)
E(X)
α/(α+β)
Var(X)
αβ/((α+β)²(α+β+1))
Support
[0, 1]

The Beta distribution is defined on [0,1] — perfect for modeling probabilities, proportions, and rates. Controlled by two shape parameters: α (think "successes + 1") and β (think "failures + 1"). Beta(1,1) = Uniform. As α,β → ∞, it concentrates around α/(α+β).

f(x) = x^(α−1) · (1−x)^(β−1) / B(α,β)     E(X) = α/(α+β)

In Bayesian inference, Beta is the conjugate prior for the Binomial: if you start with Beta(α,β) as your prior on p, and observe k successes in n trials, the posterior is Beta(α+k, β+n−k). This is why Beta is critical in Bayesian A/B testing.

from scipy.stats import beta
import numpy as np

# ─── Beta Distribution ───────────────────────────────────────────
print("═══ Beta Distribution Shapes ═══")
print(f"{'Shape':<25} {'E(X)':<10} {'Var(X)':<12} {'Description'}")
print("-"*75)
shapes = [
    ((1,1),  "Uniform — no info"),
    ((2,2),  "Symmetric, peaks at 0.5"),
    ((5,2),  "Right-skewed (high success)"),
    ((2,5),  "Left-skewed (high failure)"),
    ((0.5,0.5),"U-shaped (bimodal)"),
    ((10,10),"Narrow peak at 0.5"),
    ((20,5), "Concentrated near 0.8"),
]
for (a,b), desc in shapes:
    d = beta(a, b); e = d.mean(); v = d.var()
    print(f"Beta({a},{b}){'':<{22-len(str(a))-len(str(b))-7}} {e:<10.4f} {v:<12.6f} {desc}")

print()
# BAYESIAN A/B TESTING — the most important use case
print("═══ Bayesian A/B Test with Beta-Binomial ═══")
print("Prior: Beta(1,1) — uniform, no prior knowledge of CTR")
print("Control: 1000 visitors, 52 clicks")
print("Treatment: 1000 visitors, 68 clicks")
print()

# Prior: Beta(1,1) = uniform (no prior knowledge)
alpha_prior, beta_prior = 1, 1

# Posterior = Beta(α + successes, β + failures)
control_post   = beta(alpha_prior + 52,   beta_prior + (1000-52))
treatment_post = beta(alpha_prior + 68,   beta_prior + (1000-68))

print(f"Control posterior:   Beta({1+52}, {1+948}) → E(p) = {control_post.mean():.4f}, 95% CI: [{control_post.ppf(0.025):.4f}, {control_post.ppf(0.975):.4f}]")
print(f"Treatment posterior: Beta({1+68}, {1+932}) → E(p) = {treatment_post.mean():.4f}, 95% CI: [{treatment_post.ppf(0.025):.4f}, {treatment_post.ppf(0.975):.4f}]")

# Monte Carlo: P(treatment > control)
np.random.seed(42)
ctrl_samples  = control_post.rvs(100_000)
treat_samples = treatment_post.rvs(100_000)
p_treatment_wins = np.mean(treat_samples > ctrl_samples)
print(f"\nP(treatment CTR > control CTR) = {p_treatment_wins:.4f}")
print(f"Expected lift = {(treatment_post.mean()-control_post.mean())/control_post.mean()*100:.1f}%")

In data science: Beta is the cornerstone of Bayesian A/B testing, modeling conversion rates, click-through rates, and any bounded proportion. It's also used to model random effects in hierarchical models and as a flexible distribution for any proportion between 0 and 1.

Continuous · Generalized Wait Time

Gamma Distribution

Generalization of Exponential — waiting for k events
Notation
X ~ Γ(α,β)
E(X)
α/β
Var(X)
α/β²
Support
(0, +∞)

Gamma is a flexible positive-valued distribution that generalizes Exponential. With shape α and rate β: Gamma(1,β) = Exponential(β). When α is an integer, Gamma(α,β) is the distribution of the sum of α independent Exponential(β) variables — i.e., waiting time until the α-th event. Chi-Squared is also a special case: χ²(k) = Gamma(k/2, 1/2).

f(x) = β^α · x^(α−1) · e^(−βx) / Γ(α)     x > 0     E(X)=α/β, Var(X)=α/β²

08
Continuous · Hypothesis Testing

Chi-Squared (χ²) Distribution

Sum of squared standard normals — backbone of goodness-of-fit tests
Notation
X ~ χ²(k)
Parameter
k (degrees of freedom)
E(X)
k
Var(X)
2k
Relation
Γ(k/2, 1/2)

χ²(k) is the distribution of the sum of k independent squared Standard Normal variables. Right-skewed for small k, approaches Normal for large k. The degrees of freedom k controls the shape — small k is very skewed; large k approaches Normal.

χ²(k) = Z₁² + Z₂² + ... + Zₖ²    where Zᵢ ~ N(0,1) independent

Used in three key tests: goodness-of-fit (does data match a distribution?), independence (are two categorical variables independent?), and variance tests (is a population variance equal to some value?).

from scipy.stats import chi2, norm, chisquare, chi2_contingency
import numpy as np

# ─── Chi-Squared Distribution ────────────────────────────────────
print("═══ Chi-Squared(k) — properties across degrees of freedom ═══")
print(f"{'k (df)':<10} {'E(X)':<8} {'Var(X)':<10} {'P(X>3.84)':<14} {'95th pctile'}")
print("-"*55)
for k in [1,2,3,5,10,20,50]:
    d = chi2(k)
    print(f"{k:<10} {k:<8} {2*k:<10} {1-d.cdf(3.84):<14.4f} {d.ppf(0.95):.4f}")

print()
# TEST 1: Chi-square goodness-of-fit test
print("═══ Test 1: Goodness-of-Fit — Is this die fair? ═══")
observed = np.array([15, 18, 22, 19, 16, 10])  # 100 rolls
expected = np.array([100/6] * 6)               # fair die: 16.67 each
chi2_stat, p_gof = chisquare(observed, expected)
print(f"Observed: {observed.tolist()}")
print(f"Expected: {[round(e,1) for e in expected.tolist()]}")
print(f"χ² statistic = {chi2_stat:.4f}")
print(f"p-value      = {p_gof:.4f}")
print(f"Conclusion: {'Die is NOT fair (reject H0)' if p_gof<0.05 else 'No evidence die is unfair (fail to reject H0)'}")
print()

# TEST 2: Independence test (contingency table)
print("═══ Test 2: Independence — Gender × Product preference ═══")
table = np.array([[50, 30],   # Men:   50 prefer A, 30 prefer B
                   [20, 60]])  # Women: 20 prefer A, 60 prefer B
chi2_s, p_ind, dof, exp = chi2_contingency(table)
print(f"Contingency table:\n{table}")
print(f"χ² = {chi2_s:.4f}, df = {dof}, p = {p_ind:.6f}")
print(f"Conclusion: {'DEPENDENT — gender affects preference' if p_ind<0.05 else 'INDEPENDENT'}")

09
Continuous · Small Samples

Student's t-Distribution

Like Normal but with heavier tails — essential for small samples
Notation
X ~ t(ν)
E(X)
0 (for ν>1)
Var(X)
ν/(ν−2) for ν>2
As ν→∞
→ Normal(0,1)

When you estimate a mean from a small sample and don't know the true population standard deviation (which is almost always), the test statistic follows a t-distribution, not Normal. It's like Normal with heavier tails — accounting for the extra uncertainty from estimating σ from data. As ν (degrees of freedom) increases, t → Normal.

t = (X̄ − μ₀) / (s / √n) ~ t(n−1)     where s is the sample std dev

Rule of thumb: use t when n < 30 (or when σ is unknown, which is almost always).

from scipy.stats import t, norm, ttest_1samp, ttest_ind
import numpy as np

# ─── Student's t-Distribution ────────────────────────────────────
print("═══ t vs Normal: heavier tails for small df ═══")
print("P(|X| > 2.0) — probability in both tails beyond ±2:")
print(f"{'df (ν)':<12} {'P(|t|>2.0)':<18} {'vs Normal'}")
Z = norm(0,1)
normal_tail = 2*(1-Z.cdf(2.0))
print(f"{'Normal':<12} {normal_tail:<18.6f} (baseline)")
for df in [3,5,10,20,30,100,1000]:
    tail = 2*(1-t(df).cdf(2.0))
    ratio = tail/normal_tail
    print(f"{df:<12} {tail:<18.6f} {ratio:.2f}x heavier")
print()

# One-sample t-test
print("═══ One-Sample t-test ═══")
print("Claim: average delivery time = 30 min. Sample of 15 deliveries:")
np.random.seed(42)
sample = np.random.normal(33, 6, 15)  # true mean is 33, not 30
t_stat, p_val = ttest_1samp(sample, popmean=30)
print(f"Sample mean = {sample.mean():.2f} min, std = {sample.std():.2f}")
print(f"t-statistic = {t_stat:.4f}")
print(f"p-value     = {p_val:.4f}")
print(f"Conclusion: {'Reject H0 — mean is NOT 30 min' if p_val<0.05 else 'Fail to reject H0'}")
print()

# 95% confidence interval using t-distribution
print("─── 95% Confidence Interval using t ───")
n = len(sample)
se = sample.std(ddof=1) / n**0.5
t_crit = t(n-1).ppf(0.975)
ci_lo = sample.mean() - t_crit * se
ci_hi = sample.mean() + t_crit * se
print(f"n={n}, mean={sample.mean():.2f}, SE={se:.2f}, t_critical={t_crit:.4f}")
print(f"95% CI: [{ci_lo:.2f}, {ci_hi:.2f}]")
print(f"Does CI include 30? {'YES' if ci_lo <= 30 <= ci_hi else 'NO — 30 is outside, confirms rejection'}")

Continuous · Variance Ratios

F-Distribution

Ratio of two chi-squared variables — ANOVA and regression F-tests
Notation
X ~ F(d₁,d₂)
E(X)
d₂/(d₂−2)
Relation
χ²(d₁)/d₁ ÷ χ²(d₂)/d₂
Support
[0, +∞)

The F-distribution is the ratio of two scaled chi-squared variables. It arises naturally when comparing variances. In ANOVA, the F-statistic tests whether group means differ: F = (variance between groups) / (variance within groups). In regression, F tests whether the model explains significant variance.

F = (χ²(d₁)/d₁) / (χ²(d₂)/d₂)     F(1,ν) = t(ν)²     large F → reject H0

import numpy as np

print("'How large must F be to reject H0?' Varies with both df values")
print(f"{'d1\\d2':<8}", end='')
for d2 in [10,20,30,60,120]:
    print(f"{d2:<10}", end='')
print()
print("-"*58)
for d1 in [1,2,3,5,10]:
    print(f"{d1:<8}", end='')
    for d2 in [10,20,30,60,120]:
        print(f"{f_dist(d1,d2).ppf(0.95):<10.4f}", end='')
    print()
